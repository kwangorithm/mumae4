## 3. T값 (회차) 계산 원리

T값은 **현재 투자 비중이 전체 시드의 몇 회차만큼 들어갔는지**를 나타냅니다.

```
T값 = (보유수량 × 평단가) / 1회차금액

예시:
  보유수량 = 50주
  평단가   = $67.00
  1회차    = $168

  T값 = (50 × 67) / 168 = 3,350 / 168 ≈ 19.9
```

### T값 구간별 의미

| T값 범위 | 상태 | 설명 |
|---------|------|------|
| 0 | 신규 | 보유 없음, 새 출발 |
| 0 ~ split/2 (20) | 전반전 | 분할매수 공격 중 |
| split/2 ~ split-1 (39) | 후반전 | 별값 위주 매수 전환 |
| split-1 ~ split (39~40) | 마지막 회차 | 추가 매수 중단, 익절 대기 |
| > split (40) | 리버스 진입 | 매수 불가, 청산 개시 |

## 4. 별값(Star Price) 계산

별값은 **쿼터 익절(25%) 매도의 목표가**이며, T값이 높아질수록 목표 수익률이 낮아집니다.

```python
감가율 = 2.0 / split  # = 2/40 = 0.05 (5%)

별값비율 = 목표수익률 × (1 - 감가율 × T값)
별값 = 평단 × (1 + 별값비율)
```

### 예시 (SOXL, 목표수익률 12%, Split 40)

| T값 | 별값비율 | 평단 $67 기준 별값 |
|-----|---------|-------------------|
| 0 | 12.0% | $75.04 |
| 5 | 10.5% | $74.04 |
| 10 | 9.0% | $73.03 |
| 20 | 6.0% | $71.02 |
| 30 | 3.0% | $69.01 |
| 39 | 0.3% | $67.20 |

> T값이 높을수록 별값이 평단에 가까워져 더 쉽게 익절됩니다.

## 6. 상태별 매매 행동 규칙

### 6-1. 새 출발 (qty == 0)

- **조건**: 보유 주식 없음
- **주문**: `현재가 × 1.15 - $0.01` 로 LOC 매수
- **목적**: 장 마감가(LOC) 기준 시장가 근방에서 첫 매수

```
매수가 = ceil(현재가 × 1.15) - 0.01
매수량 = floor(1회차금액 / 매수가)
주문타입 = LOC (장 마감 지정가)
```

---

### 6-2. 전반전 (T값 < split/2)

매수 주문이 **평단 + 별값** 두 갈래로 분산됩니다.

```
절반 금액 = 1회차금액 × 0.5

평단매수:
  가격 = min(ceil(평단) - $0.01, 별값 - $0.01)
  수량 = floor(절반금액 / 평단매수가)
        + 부족분 보충 (총 N주 보장)

별값매수:
  가격 = 별값 - $0.01
  수량 = floor(절반금액 / 별값매수가)
```

---

### 6-3. 후반전 (T값 >= split/2)

별값 위주로 전환, **단일 주문**으로 단순화.

```
별값매수:
  가격 = 별값 - $0.01
  수량 = floor(1회차금액 / 별값매수가)
```

---

### 6-4. 매도 주문 (항상 병행 실행)

보유 중이라면 매수와 동시에 **쿼터 익절 + 목표 익절** 2종 매도 주문 발행.

```
쿼터 매도 (LOC):
  수량 = ceil(총보유량 / 4)      ← 25% 분할 익절
  가격 = 별값 (star_price)

목표 매도 (LIMIT):
  수량 = 총보유량 - 쿼터수량     ← 나머지 75%
  가격 = 목표가 (avg × (1 + 목표수익률))
```

---

### 6-5. 줍줍 주문 (보너스 주문)

평단보다 낮은 가격대에 **1주씩 5개** 추가 매수 주문.

```
기준수량 N = floor(1회차금액 / 평단가)

줍줍(i) = floor(1회차금액 / (N + i))  (i = 1 ~ 5)
가격 상한 = min(줍줍(i), 평단 - $0.01)
```

> 예: N=10, 1회차=$168, 평단=$16.5
> - 줍줍(1): $168/11 = $15.27
> - 줍줍(2): $168/12 = $14.00
> - 줍줍(3): $168/13 = $12.92

## 7. 리버스 모드 (청산 페이즈)

### 진입 조건

V14/V17 버전에서 아래 중 하나 해당 시 자동 진입:

- `T값 > (split - 1)` → 분할 횟수를 모두 소진
- `보유량 > 0 AND 현금 < 1회차금액` → 더 이상 매수 불가

단, **목표가 이미 달성(대박)**이면 리버스 감옥 없이 바로 익절 페이즈.

### 진입 시 exit_target 설정

```
현재 수익률 >= 기본 청산 목표:  exit_target = 0.0 (수익 청산)
현재 수익률 < 기본 청산 목표:  exit_target = 기본 청산 목표

기본 청산 목표:
  TQQQ: -15%
  SOXL: -20%
```

### 리버스 1일차 (긴급 수혈 포함)

```
┌─────────────────────────────────────────┐
│ 상황: rev_day == 1  OR  현금 < 현재가   │
│                                         │
│ 행동: 즉시 MOC (시장가 장마감) 매도     │
│       매도량 = max(4, floor(보유량/10)) │
│       (4주 미만이면 전량 청산)          │
└─────────────────────────────────────────┘
```

### 리버스 2일차 이후

```
┌─────────────────────────────────────────────────────┐
│ 별값매수 (LOC):                                      │
│   가격 = 5일이동평균 - $0.01 (또는 평단 - $0.01)    │
│   금액 = 에스크로 / 4                               │
│                                                     │
│ 별값매도 (LOC):                                      │
│   가격 = 5일이동평균 (star_price)                   │
│   수량 = max(4, floor(보유량/10))                   │
│                                                     │
│ 리버스 줍줍 (5개):                                  │
│   별값매수가보다 더 낮게 1주씩 추가 매수 대기       │
└─────────────────────────────────────────────────────┘
```

### 리버스 에스크로 (Escrow)

리버스 진입 시 매도 대금은 **에스크로 캐시**에 별도 적립되며,
리버스 기간 동안 1회차 금액 대신 `에스크로 / 4`가 매수 예산으로 사용됩니다.

## 8. 터보 모드

### 개요

급락 시 **추가 가속 매수 주문**을 코어 주문 외 1개 더 발행합니다.

### 조건

```
터보 모드 ON AND 마지막 회차가 아님 AND 현금 충분
```

### 터보 매수 가격

```
기준가 = min(평단, 전일종가)
터보가 = ceil(기준가 × 0.95) - $0.01

단, 터보가 상한 = min(터보가, 별값 - $0.01)
```

즉, 전일 종가 또는 평단 대비 **약 5% 하락 지점**에서 선매수.

## 9. V17 스나이퍼 모드 (시크릿 모드)

V17 버전 + 시크릿 모드 활성화 시 동작하는 실시간 감시 엔진.

### 하방 스나이퍼 매수 (Intercept)

```
감시 시간: 04:01 ~ 장마감 15분 전

동적 방어선 = 전일종가 × (1 - sniper_pct%)
  SOXL: SOXX 인덱스 기반 동적 계산 (기본 9%)
  TQQQ: QQQ  인덱스 기반 동적 계산 (기본 5%)

트리거 조건: 현재가 <= 동적 방어선 AND 방어선 < 평단

실행:
  1. 기존 LOC 매수 주문 취소
  2. 호가 기준 즉시 시장가 지정가 매수 (3회 시도)
  3. 체결 성공 시 당일 하방 스나이퍼 잠금
  4. 실패 시 1시간 쿨타임 후 일반 LOC 주문 복구
```

### 상방 스나이퍼 매도 (Jackpot)

```
트리거 조건: 현재가 >= 목표가

실행:
  1. 기존 매도 주문 전량 취소
  2. 매도 호가 기준 즉시 전량 매도 시도 (3회)
  3. 체결 성공 시 당일 상방 스나이퍼 잠금
  4. 졸업(수익 정산) 처리
```

### V17 스마트 방어 매수

쿼터 익절 성공 후 **스마트 core/bonus 주문**이 추가로 장전됩니다.

```
스마트 방어(평단):
  가격 = min(ceil(평단) - $0.01, 별값 - $0.01)
  수량 = N (1회차 기준 수량)
  타입 = LOC

스마트 줍줍 (5개):
  N+1 ~ N+5 수량 기준 더 낮은 가격대 1주씩
```

## 12. 안전장치 및 방어 로직

### 12-1. 워시세일 방지 방패 (Wash-Trade Shield)

매수/매도 주문 간 가격 충돌을 자동으로 교정합니다.

```
규칙 1: 매도 MOC/MOO 주문 존재 시, BUY LOC/MOC 주문 제거
규칙 2: 매수가 >= 최소 매도가인 경우, 매수가를 (최소매도가 - $0.01)로 교정
규칙 3: 모든 주문 가격 최소값 = $0.01 (마이너스 호가 방지)
```

### 12-2. T값 폭주 감지

```
T값 > split × 1.1 (110%)인 경우:
  → 🚨T값폭주(역산경고) 표시
  → 수동 매수 또는 시드 설정 오류 의심
```

### 12-3. 현금 잠금 (Locked Cash)

멀티 종목 운용 시 **다른 종목이 사용 예정인 현금**을 미리 차감합니다.

```
실제 가용 현금 = 전체 잔여 현금 - 다른 종목들의 에스크로 합계
```

### 12-4. 일일 거래 잠금 (Trade Lock)

같은 날 이미 실행된 주문 유형은 중복 실행되지 않습니다.

```
잠금 키 형식: YYYY-MM-DD_TICKER_시장타입
예: "2025-03-31_SOXL_REG"
```

## 13. 복리 (Compound) 설정

졸업(전량 익절) 성공 시 수익금 일부를 다음 시드에 자동 추가합니다.

```
추가 시드 = floor(순이익 × 복리율)

예:
  순이익 $500, 복리율 70%
  → 추가 시드 = $350
  → 다음 시드 = 기존시드 + $350
```

> 복리율 = 0%로 설정하면 복리 없이 고정 시드 유지.

## 부록: 핵심 공식 요약

```
1회차 금액    = 시드 / 분할수
T값           = (보유량 × 평단) / 1회차금액
감가율        = 2 / 분할수
별값비율      = 목표수익률 × (1 - 감가율 × T값)
별값          = ceil(평단 × (1 + 별값비율))
목표가        = ceil(평단 × (1 + 목표수익률))
터보가        = ceil(min(평단, 전일종가) × 0.95) - 0.01
줍줍(i)가     = floor(1회차금액 / (N + i))  (N = floor(1회차/평단))
리버스 매도량 = max(4, floor(보유량 / (10 or 20)))
```
